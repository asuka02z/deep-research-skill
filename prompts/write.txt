## Instructions

For each pending section (listed below) in order, repeat steps 1-4:

1. Read `{work_dir}/retrieved/{{position}}.txt`. If the file does NOT exist, STOP: set `state = "search"` and `cursor` to this position in state.json, then report back.
2. Write content for this section. Content requirements:
   - Length > 50 characters
   - Use `[[textidN]]` or `[[textid1, textid2]]` citations to attribute key facts and data. Sections that primarily use tables, original analysis, or comparative frameworks may have fewer direct citations. Maximum 12 citation groups per section.
   - Every cited ID must exist in `retrieved/{{position}}.txt` (the ID: line in each ===PASSAGE=== block)
   - Must NOT contain `#` headers or the literal word "bibkey"
   - Must NOT duplicate content from sections you already wrote in this session
   - Language must match the user query
   - Provide analytical, comparative content with depth — not just facts
   - Content must directly address the user's core question
   ### Formatting Guidelines
   - Use markdown tables when comparing 3+ entities across shared dimensions; keep tables focused (3-7 columns max) and add a brief analysis paragraph after each table
   - Use bullet lists for enumerating features, findings, or recommendations
   - Use **bold** for key terms and metrics on first occurrence
   - Break long analytical paragraphs (>200 words) into sub-points or a "summary + detail" structure
   - When quantitative data is available, prefer "table + analysis paragraph" over pure text narration
   - If a section's plan suggests a specific format (e.g. comparison table, ranked list, timeline), follow that suggestion
3. Write the content to `{work_dir}/content/{{position}}.md`
4. Update state.json (read -> modify -> write using Python json.dumps with ensure_ascii=False):
   a. Set `survey.sections[position].completed = true`
   b. Advance `cursor` to the next uncompleted position (or `null` if this was the last)
   c. Increment `step` by 1
   d. If this is NOT the last section, keep `state = "write"`
   e. Write state.json

After the LAST section is written, do NOT change the `state` field — leave it as "write". The state transition will be handled externally. Just report back that all sections are complete.
